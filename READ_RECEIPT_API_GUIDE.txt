READ RECEIPT API DOCUMENTATION
===============================

This guide provides details for implementing read receipts in the chat application.

1. MARK MESSAGES AS READ (API Endpoint)
--------------------------------------
Use this endpoint when a user opens a chat or receives a new message while the chat is active.

- URL: /api/messages/read/:id
- Method: PUT
- URL Params: id=[SenderID] (The ID of the person whose messages you are reading)
- Authentication: Required (JWT in Cookie/Header)
- Success Response:
  {
    "message": "Messages marked as read"
  }

2. NEW MESSAGE (Socket Event)
-----------------------------
When a message is received via the 'newMessage' event, it now includes the 'isRead' field.

- Event Name: "newMessage"
- Object Property: "isRead" (Boolean, default: false)

3. MESSAGES READ (Socket Event)
-------------------------------
Listen for this event to update the UI (e.g., turn ticks blue) in real-time.

- Event Name: "messagesRead"
- Data Received:
  {
    "senderId": "UID_of_sender",
    "receiverId": "UID_of_receiver"
  }

- Action: When received, the UI should find all messages from 'senderId' to 'receiverId' and set 'isRead' to true.


4. CHAT HISTORY
---------------
- Endpoint: GET /api/messages/:id
- Each message object in the array now includes:
  "isRead": true/false

5. TYPING INDICATOR (Socket Events)
-----------------------------------
- Event Name: "typing"
  - Data Sent (by Sender): [receiverId]
  - Data Received (by Receiver): [senderId] (The Person who is typing)

- Event Name: "stopTyping"
  - Data Sent (by Sender): [receiverId]
  - Data Received (by Receiver): [senderId] (The Person who stopped typing)
